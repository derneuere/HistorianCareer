# Extracting golden fixtures from a Sims 4 install

For the v0.1 test suite we use **synthetic goldens** generated by
`scripts/synthesize-goldens.mjs`. These exercise the `@s4tk/models` read/write
loop with our schema layouts and catch regressions in our pipeline, but they
do not validate byte-equality against EA's actual game data.

To upgrade to **real EA goldens**, a maintainer with a Sims 4 install can
populate `test/golden/<className>/<resourceName>.{tuning.xml,simdata}` by
hand. The recipe:

## Option A — Sims 4 Studio

1. Open Sims 4 Studio's Tools → Extract Tuning.
2. Filter by tuning class (e.g. "Career") and pick a representative resource
   like `career_Adult_Astronaut`.
3. Save the tuning XML to
   `test/golden/Career/career_Adult_Astronaut.tuning.xml`.
4. Right-click the tuning in S4S's Game File Cruiser → Export SimData.
5. Save the SimData binary to
   `test/golden/Career/career_Adult_Astronaut.simdata`.
6. Repeat for each of the 9 classes in scope.

## Option B — `@s4tk/extraction` (programmatic, future)

The `scripts/extract-golden.mjs` script is a stub that documents the intended
workflow. To enable:

1. `npm install --save-dev @s4tk/extraction`.
2. Fill in the extraction loop where the script's TODO comment is.
3. Run:
   ```bash
   node scripts/extract-golden.mjs \
     --game-dir "C:/Program Files (x86)/Origin Games/The Sims 4" \
     --class Career --class Trait --class Aspiration
   ```

## What the test suite expects

When real goldens are present at `test/golden/<className>/<resourceName>.{tuning.xml,simdata}`,
the integration test should be augmented to:

1. Load the tuning XML.
2. Build SimData via `buildSimDataForTuning(tree, ctx)`.
3. `Buffer.equals(actual, expectedBuffer)` — byte-identical match.

When this comparison fails, the `diffSimData(actual, expected)` helper
(mentioned in the plan §7; not yet implemented) will be the debugging tool of
last resort. Until that lands, parse both buffers with
`@s4tk/models`'s `SimDataResource.from()`, compare their `schemas[0]`
column-by-column, and compare each instance's `row[colName]` cell-by-cell.

## Why we don't ship EA goldens

We do not commit EA-extracted binaries to the repo for two reasons:

1. **Licensing.** EA's game data is not redistributable. Each maintainer
   extracts from their own copy.
2. **Patch drift.** EA changes schemas occasionally. A golden frozen in our
   repo would go stale; one extracted from your own install matches the patch
   you ship for.
